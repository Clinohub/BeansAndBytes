Java Conventional naming style is camelCase with full words

boolean values for java are: false and true

^: the XOR operator evaluates true for opposite values otherwise false i.e
true ^ true = false
false ^ false = false
true ^  false = true and viceversa

Don't use double or floats for Financial operations , but rather BigDecimal(not a primitive Data type)

No assignment operator in if conditions in Java
No integer in if conditions in Java

case in SwitchCase can take in multiple constants, as it iterates through each, i.e:
	char ch;
	Switch (ch) {
		case 'a','b','c','d','e':
		case 'A','B','C','D','E': return false;
		default: return true;
		}

Java Ternary operation
    var = (Condition ? valueIfTrue : valueIfFalse)

    if (Condition) {
         var = valueIfTrue;
	 }
    else {
         var = valueIfFalse
	 }
